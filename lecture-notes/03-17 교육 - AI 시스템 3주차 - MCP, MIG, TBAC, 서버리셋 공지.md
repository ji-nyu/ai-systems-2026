## 주요 내용

### 1. 수업·과제 운영

- PR 기반 제출 → 리뷰 → 수정·재제출, AI 도구 사용 내역 README에 기록
- 이론(온라인) + 실습(오프라인/리포지터리) 분리 운영

### 2. GPU 자원 공유: CPU/GPU/MIG

- GPU는 연산+메모리 동시 고려 필수, 모델이 GPU 메모리에 상주해야 동작
- **MIG**: 대형 GPU를 하드웨어 수준 고정 분할 → 동시 사용 충돌 감소
- 실무에서 MIG 적용이 적은 이유: 설정 변경 시 GPU 리셋(루트 권한), 세션 중단 위험
- 쿠버네티스 연계 MIG 파드 스케줄링 가능 — 개념 이해 필수

### 3. 현대 AI 개발 스택

- API(내부 은닉) vs 인터페이스(채널 틀) vs 프로토콜(형식 규약) 구분
- LLM + DB/Slack/GitHub/JIRA 연동 필수, 외부 서비스 형식 맞춤에 시간·비용 소요

### 4. MCP (Model Context Protocol)

- Anthropic의 LLM↔외부 리소스 상호작용 표준화 프로토콜
- JSON 기반, 로컬(stdio)/네트워크(streamable) 모두 지원
- HOTL/HITL 통제 모델과 결합해 안전 운영
- 수업: 개념 인지 + 필수 요구사항 구현

### 5. 통제 모델과 보안

- **HOTL**: 단계 승인·중지로 통제 유지
- **HITL**: 사용자 입력 강제로 모호성 제거 (오픈클로버 사례: "깔끔하게" → 전부 삭제)
- **TBAC**: 태스크 내용·위험도 기준 권한 부여 (전통 RBAC의 AI 환경 한계 극복)
- **프롬프트 인젝션**: 자연어 내 악의적 지시 → SQL 인젝션 유사, 입력 처리·내용 통제 핵심
- **MCP Gateway**: 태스크 위험 평가·인가 하네스 계층, 백엔드 API는 자원 통제만 담당

### 6. 서버 운영 공지 (DGX H100)

- **3/19 유저 계정 초기화/리셋 예정** — NVIDIA OS 재설치 불가피
- RAID에 데이터 보존 가능하나 **권한 승계 불확실** → 사전 백업 필수
- 도커 루트 리소스 접근 문제 → sudo 도커로 정리 필요
- 홈 디렉토리 접근: Google Drive·MCP 연계로 보완

### 7. 비용·구독·팀 운영
